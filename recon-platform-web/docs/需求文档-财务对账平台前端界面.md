## 1. 项目概述

### 1.1 产品定位

财务对账平台是一个**智能化企业级对账系统**，面向企业财务人员，支持ERP系统与银行/第三方支付等多数据源之间的自动对账。平台融合AI能力，提供语义匹配、差异自动分类、根因分析、自然语言查询等智能化功能。

### 1.2 用户角色

| 角色 | 编码 | 权限范围 |
|------|------|---------|
| 超级管理员 | SUPER_ADMIN | 全部功能 + 系统管理 |
| CFO | CFO | 仪表盘 + 分析 + 审批 |
| 财务经理 | FINANCE_MANAGER | 对账任务 + 差异管理 + 审批 |
| 会计 | ACCOUNTANT | 对账任务 + 差异处理 |
| 审计员 | AUDITOR | 查看所有 + 审计日志 |

### 1.3 设计目标

- **高效**: 一键执行对账、批量AI分类、批量审核
- **智能**: AI辅助匹配打分、自动分类差异、根因分析、自然语言查询
- **可追溯**: 完整的审计日志、操作记录
- **可视化**: 仪表盘图表、趋势分析、健康度监控

### 3.2 仪表盘

**路由**: `/dashboard`  
**文件**: `src/views/dashboard/Dashboard.vue`

#### 3.2.1 统计卡片区（4个指标卡片）

| 卡片 | 指标 | 数据来源 |
|------|------|---------|
| 对账任务 | 总任务数 + 运行中数量 | `GET /api/v1/analytics/dashboard` → `taskStats` |
| 自动匹配率 | 百分比 + 已匹配总数 | `GET /api/v1/analytics/dashboard` → `matchStats` |
| 差异数量 | 差异总数 + SLA超期数(红色高亮) | `GET /api/v1/analytics/dashboard` → `discrepancyStats` |
| 差异解决率 | 已解决/总计 × 100% | 前端计算 |

每个卡片包含:**彩色图标** + 标签 + 主数值 + 辅助说明文字

#### 3.2.2 图表区域

**左侧 — 对账趋势图** (占 2/3 宽度)
- 图表类型: 柱状图 + 折线图组合
- 维度: 任务数(柱)、匹配率%(折线)、差异数(柱)
- X轴: 月份, Y轴左: 数量, Y轴右: 百分比
- 支持切换近6月 / 近12月
- 数据来源: `GET /api/v1/analytics/trend`

**右侧 — 差异分类分布饼图** (占 1/3 宽度)
- 图表类型: 环形饼图
- 维度: 7种差异类别分布 (TIME_DIFF/FEE_DIFF/EXCHANGE_DIFF/HUMAN_ERROR/UNREACHED/DUPLICATE/UNKNOWN)
- 每个分类显示中文名称
- 数据来源: `GET /api/v1/analytics/dashboard` → `categoryDistribution`

#### 3.2.3 最近任务列表

- 表格列: 任务名称、类型(标签)、对账期间、匹配数、差异数、匹配率(进度条)、状态(标签)、完成时间
- 支持点击"查看全部"跳转至对账工作台

#### 3.2.4 状态映射

| 字段 | 值 → 显示 |
|------|----------|
| 任务类型 | BANK→银行 / THIRD_PAYMENT→第三方 / AR→应收 / AP→应付 |
| 任务状态 | PENDING→待执行(info) / RUNNING→运行中(warning) / COMPLETED→已完成(success) / FAILED→失败(danger) |
| 匹配率 | ≥90%绿色 / 70-90%默认 / <70%红色 |

2. **快捷模板**:

1. **批量AI分类**: 选中多条差异 → 调用 `POST /api/v1/discrepancy/batch-classify`

4. **分配处理人**: 弹出对话框选择处理人 → 调用 `PUT /api/v1/discrepancy/{id}/assign`
5. **解决差异**: 弹出对话框选择解决方式(确认差异/创建调整分录/标记重复/忽略) + 备注 → 调用 `PUT /api/v1/discrepancy/{id}/resolve`

### 4.2 AI相关接口详细说明

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 差异分类 | POST | `/api/v1/discrepancy/{id}/classify` | 单条差异AI分类，返回包含分类结果和风险等级的差异对象 |
| 批量分类 | POST | `/api/v1/discrepancy/batch-classify` | 批量触发AI分类 |
| NL规则生成 | POST | `/api/v1/rule/generate-from-nl` | 自然语言描述 → 结构化规则配置 |
| 保存生成规则 | POST | `/api/v1/rule/save-generated` | 保存AI生成的规则 |
| 报告生成 | POST | `/api/v1/analytics/report` | AI自动撰写对账报告 |
| NL查询 | POST | `/api/v1/analytics/nl-query` | 自然语言 → Text-to-SQL → 执行 → 结果润色 |

### 4.3 对账匹配引擎四层架构

```
Layer 1: 精确匹配 → Layer 2: 规则匹配 → Layer 3: AI语义匹配 → Layer 4: 1:N拆单匹配
```

| 层级 | 说明 | 置信度 | 动作 |
|------|------|--------|------|
| L1 精确匹配 | 金额+流水号+日期完全一致 | 100 | AUTO_CONFIRMED |
| L2 规则匹配 | 基于容差配置的字段匹配 | 90-98 | AUTO_CONFIRMED |
| L3 AI语义匹配 | LLM多维度精排 | ≥85→AUTO / 70-85→PENDING_REVIEW / <70→跳过 | 视得分 |
| L4 拆单匹配 | 子集和回溯 N笔对1笔 | 85 | PENDING_REVIEW |

AI语义匹配5维度打分权重: **金额30% + 日期20% + 交易方25% + 摘要15% + 流水号10%**

## 6. AI能力集成说明

### 6.1 AI能力全景

本平台通过 `AIService` 统一接口集成以下AI能力：

1. **智能字段映射**: 自动识别源和目标字段的对应关系
2. **语义匹配**: 多维度(金额/日期/交易方/摘要/流水号)LLM精排打分
3. **差异分类**: 自动将差异分为 TIME_DIFF / FEE_DIFF / EXCHANGE_DIFF / HUMAN_ERROR / UNREACHED / DUPLICATE / UNKNOWN
4. **根因分析**: Chain-of-Thought推理，给出处理建议和调整分录
5. **自然语言规则生成**: 用中文描述匹配需求，自动生成结构化规则
6. **自然语言查询**: Text-to-SQL，将中文问题转为SQL执行并润色结果
7. **报告生成**: AI自动撰写对账报告

### 6.2 AI模型配置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `ai.llm.provider` | LLM提供商 | openai / claude / local |
| `ai.llm.default-model` | 默认模型 | gpt-4o |
| `ai.llm.simple-model` | 简单任务模型 | gpt-4o-mini |
| `ai.llm.reasoning-model` | 推理模型 | gpt-4o |
| `ai.semantic-match.embedding-threshold` | Embedding粗排阈值 | 0.75 |
| `ai.semantic-match.llm-threshold-auto` | 自动确认阈值 | 0.85 |
| `ai.semantic-match.llm-threshold-recommend` | 推荐审核阈值 | 0.70 |

### 6.3 前端AI交互模式

```
用户操作 → 前端按钮(loading状态) → API调用 → 后端AIService
    → MockAIService(开发/测试) 或 生产AI实现(Claude/GPT/本地模型)
    → 返回结果 → 前端展示(刷新列表/对话框/消息提示)
```

### 6.4 Mock模式说明

当前前端API调用层已实现Mock降级：当后端接口不可用时（如开发阶段API未就绪），会自动使用内置Mock数据，并在成功提示中标注"(Mock)"。切换到生产环境只需后端部署对应的 `productionAIService` Bean即可。
